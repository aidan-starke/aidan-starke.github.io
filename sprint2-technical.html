Classes and ID's in CSS

1 July 2020

What are the best practices associated with using classes vs. ids?

 

Before we get into using Classes and ID's, we need to understand the key differences between them.
ID's are unique. This means that for any element in a webpage, it can only have one ID. There can also only be that one element with that ID on the website.
Classes however, are not unique. There can be any number of elements on a website that are of the same class.
An element can have styling both from a class, and an ID. If any of the code clashes between these, the ID code will take priority.

 

Therefore, the best practice is as follows:
If there is one element that you wish to have styled differently from other elements on the website, this is where an ID would be used.
If you have multiple elements on the webpage which require the same styling, these can be all put in the same class.
If an element requires MOST of the styling from a class, but needs some specific adjustment, you can use both here - the code related to the ID will make the adjustments neccesary.
