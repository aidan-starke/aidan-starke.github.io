Problem Solving

15 July 2020
Story 1: A time I got blocked on a simple problem
My website was working all sweet! Locally. However, when I pushed my changes to GitHub, the images wouldn't load. I found myself at a point of frustration, but a quick google search revealed I can upload images to my repo through the GitHub website! Problem resolved.

 

Story 2: A time I solved a problem in an elegant way
When making my minesweeper, I decided to do the stretch work of creating the board using a function. I knew I would have to loop through the rows and columns of the board, but could not remember how to loop though both at once. When googling it, I recognised the term 'nested for loops', I'd done that before! I went back to my CodeCademy course and found where I'd used it, and applied the technique to my minesweeper page. I'm stoked with how the code turned out!

 

Problem solving techniques
  1. Pseudocode
    2. Pseudocode is hugely helpful for larger pieces of code. The more things going on in a piece of code, the harder it is to remember while I'm writing it. I find it really helpful to have a written down version of what I'm trying to achieve that I can go back to.

     

  2. Try something
    3. Often my first step when something breaks - I try something else! If this fails after a few tries then I move on to another technique as I find a lot of time can be wasted trying random things.

     

  3. Rubber Ducky
    4. I love the rubber ducky! Running through my code top to bottom helps me make sure that everything is in order, and I can isolate where exactly the problem is.

     

  4. Console.log
    5. It blew my mind when I discovered you could console.log to a webpage. It's perfect for testing how far through my code the webpage got before breaking, so I know exactly where I need to change something.

     

  5. Google
    6. Google has so far provided me with every answer I could want! The trick is in knowing exactly what to search for - asking google the literal question I have generally points me in the right direction.

     

  6. Asking peers/coaches for help
    7. I'm incredibly lucky to have a full-stack developer for a flatmate - it makes it really easy for me to reach out for help. I have also found help through the foundations slack, it's awesome to have such a big support crew with everyone learning and willing to help!

     

  7. Improving my process through reflection
    8. This is a big one for me - I use it everyday for self-improvement so it is natural for me to use it when coding.
